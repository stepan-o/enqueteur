## KVP-0001 v0.1 Compliance Checklist (Sprint 14.0A Locked)

### A) Envelope + Dispatch (Non-negotiable)
- [ ] Every message uses the mandatory envelope: `kvp_version`, `msg_type`, `msg_id`, `sent_at_ms`, `payload`.
- [ ] Decoder **always** decodes envelope first.
- [ ] Top-level dispatch is **only** `envelope.msg_type` (no shape-based dispatch).
- [ ] Payload contains **no** inner discriminator for top-level message selection.

### B) Versioning + Unknown Handling
- [ ] `kvp_version` MUST equal `"0.1"` or hard reject.
- [ ] `schema_version` is negotiated in handshake and MUST be supported or hard reject.
- [ ] Unknown `msg_type` → MUST reject.
- [ ] Unknown enum variants inside payloads → MUST reject.
- [ ] Unknown object fields inside recognized payload objects → MUST ignore (no `deny_unknown_fields`).

### C) Session State Machine (Strict)
- [ ] No state delivery before handshake + subscribe complete.
- [ ] Allowed sequence enforced:
    1) `VIEWER_HELLO`
    2) `KERNEL_HELLO`
    3) `SUBSCRIBE`
    4) `SUBSCRIBED`
    5) then `FULL_SNAPSHOT` and/or `FRAME_DIFF`
- [ ] If violated: kernel sends `ERROR` (and may close).

### D) Handshake Requirements
- [ ] `VIEWER_HELLO.supported_schema_versions` is non-empty.
- [ ] Kernel selects exactly one `schema_version` for session.
- [ ] `KERNEL_HELLO.render_spec` is **REQUIRED** in v0.1.
- [ ] `render_spec` MUST be stable for a run; any change implies new `run_id`.
- [ ] `render_spec.coord_system.bounds` MUST be present.

### E) Subscription Rules
- [ ] Kernel sends **no unsolicited state** (no state before `SUBSCRIBED`).
- [ ] `SUBSCRIBE.stream` MUST be `"LIVE"` (anything else rejected).
- [ ] `channels` treated as a **set**: no duplicates, non-empty.
- [ ] Kernel MUST NOT send data for **unsubscribed** channels.
- [ ] Reserved channels (e.g., `NARRATIVE`) MUST be rejected or stripped; MUST NOT produce delivery in v0.1.

### F) Baseline Snapshot Rules (AAA Fix Pack)
- [ ] If `snapshot_policy = ON_JOIN`: kernel MUST send **exactly one** baseline `FULL_SNAPSHOT` immediately after `SUBSCRIBED` and **before** any `FRAME_DIFF`.
- [ ] If `snapshot_policy = NEVER`: kernel MUST reject OR require viewer to obtain baseline via replay artifacts before diffs.

### G) Time + Tick Integrity
- [ ] Kernel is time authority; viewer never advances tick locally.
- [ ] Viewer applies diffs strictly in ascending tick order.
- [ ] Tick continuity is enforced; gaps trigger desync recovery path.

### H) `FULL_SNAPSHOT` Invariants
- [ ] Includes `schema_version`, `tick`, `step_hash`.
- [ ] Contains only kernel state for subscribed channels.
- [ ] Payload is primitives-only (Rust portable).
- [ ] Canonical ordering + float quantization applied before emit.

### I) `FRAME_DIFF` Invariants
- [ ] Includes `schema_version`, `from_tick`, `to_tick`, `prev_step_hash`, `step_hash`, `ops[]`.
- [ ] v0.1 LIVE tick mode: `to_tick == from_tick + 1`.
- [ ] `ops[]` must be applied **in order** as emitted.
- [ ] Kernel emits ops in deterministic order (or explicitly treats “as-emitted is canonical”).
- [ ] `prev_step_hash` must match expected chain (baseline or prior diff).

### J) Commands (Viewer → Kernel)
- [ ] Viewer sends commands only via `INPUT_COMMAND` (no state mutation).
- [ ] Kernel responds with `COMMAND_ACCEPTED` or `COMMAND_REJECTED` keyed by `client_cmd_id`.
- [ ] Commands are validated/sanitized and applied deterministically at `tick_target` or rejected.

### K) Replay (v0.1 ARTIFACTS Mode)
- [ ] Replay is controlled via `REPLAY_BEGIN` / `REPLAY_SEEK` (not `SUBSCRIBE.stream=REPLAY`).
- [ ] `REPLAY_BEGIN.delivery` MUST be `ARTIFACTS` (others rejected).
- [ ] In `ARTIFACTS` mode: kernel MUST NOT stream `FULL_SNAPSHOT`/`FRAME_DIFF` in-band; viewer fetches via artifacts/manifest.

### L) Desync Detection + Recovery (Mode-aware)
- [ ] Viewer can report: `TICK_GAP`, `FROM_TICK_MISMATCH`, `INVALID_PAYLOAD`, `EXTERNAL_HASH_MISMATCH`.
- [ ] LIVE: kernel `DESYNC_CONFIRMED` + MUST send `FULL_SNAPSHOT(recovery_tick)` in-band; diffs resume only after.
- [ ] ARTIFACTS replay: kernel `DESYNC_CONFIRMED` includes `recovery_tick` + pointer; viewer fetches snapshot via artifacts (no in-band state).

### M) Canonicalization + Testability (CI must enforce)
- [ ] Canonical ordering (agents/rooms/items/events) enforced.
- [ ] Float quantization default `Q1E3` applied everywhere viewer-facing.
- [ ] Golden snapshot/diff traces exist and are used for parity tests.
- [ ] Viewer replay of golden traces must be deterministic (tick chain + ordering + validation).
