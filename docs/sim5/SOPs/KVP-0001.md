# 📡 KVP-0001 — Kernel ↔ Viewer Protocol
## _Public Interface Contract for Loopforge Sim5 → SimX_

**Status:** Draft v1.0  
**Audience:** Kernel, Viewer, Tooling, Replay, QA  
**Scope:** Deterministic simulation → external renderers  
**Applies to:** Godot, Unreal Engine, Unity, future runtime shells

---

## 0. Purpose

Define a **public, versioned, testable protocol** between the  
**Loopforge deterministic simulation kernel** and any **viewer or client**.

This protocol ensures:
- engine-agnostic rendering (viewer engines are replaceable)
- deterministic replay (replay, streaming, and tooling are first-class)
- multi-viewer compatibility
- long-term forward portability (the kernel is portable across runtimes)

If it is not expressed in this protocol,  
**it is not part of the system**.

---

## 1. Core Principles

1. **Protocol, not code**
   - Viewer never imports kernel code
   - Kernel never imports viewer code
   - No shared runtime state
   - No engine-specific logic
   - No implicit contracts

2. **Kernel sovereignty**
    - Kernel is the only authority
    - All simulation state lives in the kernel
    - Viewers are observers + input emitters

3. **Determinism is sacred**
    - Replay defines truth
    - Same seed + inputs = same outputs
    - Viewer must never affect determinism
    - Visual polish is secondary

4. **Primitives-only payloads**
    - Rust-portable
    - JSON / msgpack / protobuf friendly

5. **Replaceable viewers**
    - Godot today
    - Unreal tomorrow
    - Something unknown later

6. **Testable by construction**
    - Golden traces
    - Stable schemas

---

## 2. Transport Model

KVP-0001 is **transport-agnostic**.

### Recommended Implementations
- **WebSocket** (Sim4 default): bidirectional streaming
- **TCP** (local or high-throughput environments)
- **HTTP** (snapshot + replay fetch)

Transport choice:
- MUST NOT affect message semantics
- MUST preserve message order
- MUST support bidirectional flow

Transport failures must be recoverable
via resubscription and snapshot replay.

---

## 3. Message Envelope (Mandatory)

All protocol messages use the same envelope:
```json
{
  "kvp_version": "0.1",
  "msg_type": "FULL_SNAPSHOT",
  "msg_id": "uuid",
  "sent_at_ms": 0,
  "payload": {}
}
```
**Fields**
* `kvp_version`: protocol version
* `msg_type`: message discriminator
* `msg_id`: unique message identifier
* `sent_at_ms`: sender timestamp (non-authoritative)
* `payload`: schema-defined body

Messages without this envelope are invalid.

---

## 4. Versioning & Compatibility

### Version Layers
- **KVP version** → protocol compatibility
- **Schema version** → data structure compatibility

A viewer may connect only if:

- `kvp_version.major` matches
- `schema_version` is supported by the viewer

### Compatibility Policy
- Major version mismatch → hard reject
- Minor versions may add optional fields
- Required fields are immutable
- Schema-breaking changes require a new major version

Kernel must explicitly reject unsupported viewers.

---

## 5. Kernel Identity

On connection, the kernel MUST expose:

- `engine_name` (e.g. sim5, simx)
- `engine_version` (semver or commit hash)
- `schema_version` (integration schema version)
- `world_id` (UUID)
- `run_id` (UUID per execution)
- `seed`
- `tick_rate_hz` (integer)

This identity anchors determinism and replay.

## 6. Session Handshake

### Viewer → Kernel: `VIEWER_HELLO`

```json
{
  "viewer_name": "unreal-client",
  "viewer_version": "0.1.0",
  "supported_schema_versions": ["2"],
  "supports": {
    "diff_stream": true,
    "full_snapshot": true,
    "replay_seek": true
  }
}
```

---

### Kernel → Viewer: `KERNEL_HELLO`
Contains kernel identity and run metadata.
```json
{
  "engine_name": "sim4",
  "engine_version": "0.9.0",
  "schema_version": "2",
  "world_id": "uuid",
  "run_id": "uuid",
  "seed": 12345,
  "tick_rate_hz": 30,
  "time_origin_ms": 0
}
```
Handshake must succeed before any state is sent.

## 7. Subscription Model

After handshake, viewers explicitly subscribe to data streams.

---

### Viewer → Kernel: `SUBSCRIBE`
- stream mode (LIVE / REPLAY)
- requested channels
- diff policy
- snapshot policy

```json
{
  "stream": "LIVE",
  "channels": ["WORLD", "AGENTS", "EVENTS", "NARRATIVE"],
  "diff_policy": "DIFF_ONLY",
  "snapshot_policy": "ON_JOIN",
  "compression": "NONE"
}
```

---

### Kernel → Viewer: `SUBSCRIBED`
- stream_id
- effective channels
- enforced policies

```json
{
  "stream_id": "uuid",
  "effective_channels": ["WORLD", "AGENTS", "EVENTS"],
  "effective_diff_policy": "DIFF_ONLY"
}
```
Kernel MUST NOT send unsolicited data.

---

## 8. Channels

Channels scope data delivery.

Core channels:
- WORLD
- AGENTS
- ITEMS
- EVENTS
- NARRATIVE
- DEBUG

If a channel is not subscribed,  
data from that channel MUST NOT be sent.

---

## 9. Time Model

The kernel is the sole authority on time.

### Kernel Tick Fields
- `tick` (monotonic integer)
- `step_hash` (deterministic hash)
- `sim_time_ms`: derived time (optional)

Viewers must never invent or advance time.

The viewer MUST treat ticks as immutable history.

---

## 10. Delivery Modes

- **Live Mode**: kernel streams diffs each tick
- **Replay Mode**: kernel serves historical diffs

Tick order MUST be preserved.  
Dropped ticks are invalid.

---

## 11. Full Snapshot

Snapshots provide a complete baseline.

A full snapshot allows a viewer to reconstruct state.

---

### Kernel → Viewer: `FULL_SNAPSHOT`
Includes:
- world state
- agents
- items
- events
- narrative fragments

```json
{
  "tick": 1200,
  "step_hash": "abc123",
  "world": {...},
  "agents": [...],
  "items": [...],
  "events": [...],
  "narrative_fragments": [...]
}
```

Rules:
- Snapshot MUST be complete
- primitives-only
- canonical ordering
- quantized floats
- schema-versioned

Snapshots define canonical truth and baseline state.

---

## 12. Frame Diff Stream

Diffs describe transitions between ticks.

### Kernel → Viewer: `FRAME_DIFF`
Includes:
- from_tick
- to_tick
- step_hash
- diff body

Diffs MUST:
- be applied in order
- assume a valid baseline
- be replayable without loss

```json
{
  "from_tick": 1200,
  "to_tick": 1201,
  "step_hash": "def456",
  "diff": {
    "agent_moves": [...],
    "agent_spawns": [...],
    "agent_despawns": [...],
    "room_replacements": [...],
    "event_replacements": [...],
    "narrative_replacements": [...]
  }
}
```

---

### Diff Policies

Supported policies:
- `DIFF_ONLY`
- `PERIODIC_SNAPSHOT`
- `SNAPSHOT_ON_DESYNC`

Policy choice affects delivery,  
not simulation semantics.

---

## 13. Viewer → Kernel Input

Viewers send **commands**, not state.

### `INPUT_COMMAND`
Includes:
- client_cmd_id
- tick_target
- command payload

Commands are:
- validated
- sanitized
- logged
- deterministically applied

```json
{
  "client_cmd_id": "uuid",
  "tick_target": 1210,
  "cmd": {
    "type": "WORLD_NUDGE",
    "payload": {
      "room_id": 7,
      "nudge_type": "LIGHTS_FLICKER",
      "strength": 0.2
    }
  }
}
```

Kernel responds with:
* `COMMAND_ACCEPTED`
* `COMMAND_REJECTED`

Accepted commands are logged into history.  
Rejected commands must return reasons.

---

## 14. Input Philosophy

Sim5 supports **indirect control only**.

Allowed:
- WORLD_NUDGE
- DEBUG_PROBE
- REPLAY_CONTROL

Disallowed:
- direct movement
- state mutation
- imperative scripting

Influence the world like weather,  
not like hands.

---

## 15. Replay Control
Replay uses the **same protocol** as live play.

Replay supports:
- begin
- pause
- resume
- seek
- scrub

Replay must produce identical results  
to live execution given the same inputs.

### Viewer → Kernel: `REPLAY_BEGIN`

```json
{
  "run_id": "uuid",
  "start_tick": 0,
  "end_tick": 20000,
  "delivery": "CHUNKED"
}
```

---

### Viewer → Kernel: `REPLAY_SEEK`
```json
{
  "tick": 8500,
  "policy": "SNAPSHOT_THEN_DIFFS"
}
```
Replay MUST reproduce the same state as live execution.

---

## 16. Narrative Projection

Narrative data is viewer-facing and non-authoritative.

### Sources
1. Deterministic kernel summaries
2. Non-deterministic narrative sidecar output

### Narrative Fragment
- are tagged nondeterministic
- have TTL
- are replaceable
- never affect kernel logic

```json
{
  "entity_id": 12,
  "kind": "INNER_MONOLOGUE",
  "text": "I hate this plaza.",
  "ttl_ticks": 120,
  "nondeterministic": true
}
```

**Rules**
* Narrative MUST NOT affect kernel logic
* Nondeterministic fragments MUST be tagged
* Kernel state derives only from numeric components

---

## 17. Desync Detection

Viewers verify determinism using `step_hash` and may report a mismatch.

### DESYNC_REPORT
On mismatch viewer sends `DESYNC_REPORT`:

```json
{
  "tick": 1201,
  "expected_step_hash": "def456",
  "actual_viewer_hash": "zzz999"
}
```

### Kernel response
Kernel responds with:
* `DESYNC_CONFIRMED` + `FULL_SNAPSHOT`
* or `DESYNC_DENIED`

Desync recovery is protocol-defined,  
not viewer-defined.

Errors include:

Errors terminate sessions explicitly.

```rust
//! KVP-0001 — Kernel ↔ Viewer Protocol (Rust traits)
//! Draft v0.1 — primitives-only, transport-agnostic, testable.
//!
//! Design goals:
//! - Kernel/viewer swap without code-sharing
//! - Deterministic kernel + diff stream
//! - Versioned envelope + schema version
//! - Works with JSON now; msgpack/protobuf later

use std::fmt::Debug;

pub type Tick = u64;

/// Protocol version (KVP) and integration schema version (your IntegrationSchemaVersion).
#[derive(Clone, Debug, PartialEq, Eq)]
pub struct ProtocolVersions {
    pub kvp_version: &'static str,    // e.g. "0.1"
    pub schema_version: &'static str, // e.g. "2"
}

/// Every message rides inside this envelope.
///
/// Payload is protocol-defined but transport/encoding-agnostic.
#[derive(Clone, Debug, PartialEq, Eq)]
pub struct Envelope<P> {
    pub kvp_version: &'static str, // "0.1"
    pub msg_type: MsgType,
    pub msg_id: MsgId,
    pub sent_at_ms: u64, // non-authoritative
    pub payload: P,
}

#[derive(Clone, Debug, PartialEq, Eq, Hash)]
pub struct MsgId(pub String); // UUID string in v0.1

/// KVP-0001 message types (v0.1).
#[derive(Copy, Clone, Debug, PartialEq, Eq)]
pub enum MsgType {
    // Handshake/session
    ViewerHello,
    KernelHello,
    Subscribe,
    Subscribed,
    Unsubscribe,
    Ping,
    Pong,

    // State delivery
    FullSnapshot,
    FrameDiff,

    // Input
    InputCommand,
    CommandAccepted,
    CommandRejected,

    // Replay
    ReplayBegin,
    ReplayReady,
    ReplaySeek,
    ReplayEnd,

    // Debug/recovery
    DebugProbe,
    DebugProbeResult,
    DesyncReport,
    DesyncConfirmed,
    DesyncDenied,

    // Errors
    Warn,
    Error,
}

/// A strongly-typed view of messages.
/// Implementations typically use an enum with one variant per msg payload.
pub trait KvpMessage: Clone + Debug + Send + Sync + 'static {
    /// The message discriminator for this payload.
    fn msg_type(&self) -> MsgType;

    /// The protocol versions expected by this message (kvp + schema).
    fn versions(&self) -> ProtocolVersions;
}

/// Transport-agnostic frame interface.
///
/// WebSocket/TCP/etc implement this. No JSON assumptions here.
pub trait KvpTransport: Send {
    type Err: Debug + Send + Sync + 'static;

    /// Send raw bytes (already encoded).
    fn send(&mut self, bytes: &[u8]) -> Result<(), Self::Err>;

    /// Receive raw bytes (already encoded).
    /// Non-blocking transports may return Ok(None) when no frame is ready.
    fn recv(&mut self) -> Result<Option<Vec<u8>>, Self::Err>;
}

/// Encoding/decoding boundary.
///
/// Swap JSON for msgpack/protobuf later without changing kernel/viewer logic.
pub trait KvpCodec: Send + Sync {
    type Err: Debug + Send + Sync + 'static;

    /// The “wire” representation of any message payload set your app supports.
    type Wire: Clone + Debug + Send + Sync + 'static;

    fn encode(&self, msg: &Envelope<Self::Wire>) -> Result<Vec<u8>, Self::Err>;
    fn decode(&self, bytes: &[u8]) -> Result<Envelope<Self::Wire>, Self::Err>;
}

/// High-level handler for incoming messages.
///
/// You’ll usually implement this on the kernel side (viewer → kernel),
/// and separately on the viewer side (kernel → viewer).
pub trait KvpHandler<M>: Send
where
    M: KvpMessage,
{
    type Err: Debug + Send + Sync + 'static;

    fn on_message(&mut self, msg: Envelope<M>) -> Result<(), Self::Err>;
}

/// Optional: hook for desync recovery policies (snapshot-on-desync etc).
pub trait DesyncPolicy: Send {
    fn on_desync_report(&mut self, tick: Tick, expected: &str, got: &str);
}

/// Hashing contract: kernel provides step_hash per tick; viewer verifies.
pub trait StepHash: Send + Sync {
    /// Stable hash of the canonical state transition at this tick.
    fn step_hash(&self) -> &str;
}

/// Ordering + quantization rules are what make golden tests stable.
pub trait Canonicalize {
    /// Sort arrays, normalize float precision, enforce stable IDs, etc.
    fn canonicalize(&mut self);
}

/// --- Core domain payload traits (viewer-facing, primitives-only) ---

/// Full snapshot payload (baseline).
pub trait FullSnapshotPayload: KvpMessage + Canonicalize {
    fn tick(&self) -> Tick;
    fn step_hash(&self) -> &str;
}

/// Frame diff payload (tick-to-tick transition).
pub trait FrameDiffPayload: KvpMessage + Canonicalize {
    fn from_tick(&self) -> Tick;
    fn to_tick(&self) -> Tick;
    fn step_hash(&self) -> &str;
}

/// Viewer → Kernel input command payload.
pub trait InputCommandPayload: KvpMessage + Canonicalize {
    fn client_cmd_id(&self) -> &str;
    fn tick_target(&self) -> Tick;
}

/// Kernel response: accepted command.
pub trait CommandAcceptedPayload: KvpMessage {
    fn client_cmd_id(&self) -> &str;
}

/// Kernel response: rejected command.
pub trait CommandRejectedPayload: KvpMessage {
    fn client_cmd_id(&self) -> &str;
    fn reason(&self) -> &str;
}

/// Replay control payloads.
pub trait ReplayBeginPayload: KvpMessage {
    fn run_id(&self) -> &str;
    fn start_tick(&self) -> Tick;
    fn end_tick(&self) -> Tick;
}
pub trait ReplaySeekPayload: KvpMessage {
    fn tick(&self) -> Tick;
}

/// --- Session model traits ---

/// ViewerHello payload (capabilities + supported schema versions).
pub trait ViewerHelloPayload: KvpMessage {
    fn viewer_name(&self) -> &str;
    fn viewer_version(&self) -> &str;
    fn supported_schema_versions(&self) -> &[String];
}

/// KernelHello payload (identity + run seed + tick rate).
pub trait KernelHelloPayload: KvpMessage {
    fn engine_name(&self) -> &str;
    fn engine_version(&self) -> &str;
    fn schema_version(&self) -> &str;
    fn world_id(&self) -> &str;
    fn run_id(&self) -> &str;
    fn seed(&self) -> i64;
    fn tick_rate_hz(&self) -> u32;
}

/// Subscription payloads.
pub trait SubscribePayload: KvpMessage {
    fn stream(&self) -> &str; // "LIVE" | "REPLAY" in v0.1
    fn channels(&self) -> &[String];
    fn diff_policy(&self) -> &str; // "DIFF_ONLY" etc
    fn snapshot_policy(&self) -> &str; // "ON_JOIN" etc
}
pub trait SubscribedPayload: KvpMessage {
    fn stream_id(&self) -> &str;
    fn effective_channels(&self) -> &[String];
    fn effective_diff_policy(&self) -> &str;
}

/// --- Engine-side contracts (kernel) ---

/// The kernel produces snapshots/diffs and consumes commands.
pub trait KernelApi<S, D, C>: Send
where
    S: FullSnapshotPayload,
    D: FrameDiffPayload,
    C: InputCommandPayload,
{
    type Err: Debug + Send + Sync + 'static;

    fn build_snapshot(&mut self, tick: Tick) -> Result<S, Self::Err>;
    fn build_diff(&mut self, from: Tick, to: Tick) -> Result<D, Self::Err>;

    /// Validate + sanitize command before deterministically applying.
    fn apply_command(&mut self, cmd: &C) -> Result<(), Self::Err>;
}

/// Viewer-side contract: consumes snapshots/diffs and renders, emits commands.
pub trait ViewerApi<S, D, C>: Send
where
    S: FullSnapshotPayload,
    D: FrameDiffPayload,
    C: InputCommandPayload,
{
    type Err: Debug + Send + Sync + 'static;

    fn on_snapshot(&mut self, snap: S) -> Result<(), Self::Err>;
    fn on_diff(&mut self, diff: D) -> Result<(), Self::Err>;

    /// Optional: viewer can request a command emission (UI nudge etc).
    fn poll_outgoing_command(&mut self) -> Result<Option<C>, Self::Err>;
}

/// --- A tiny “pump” loop skeleton (transport + codec + handler) ---

pub fn pump_wire<T, C, H>(
    transport: &mut T,
    codec: &C,
    handler: &mut H,
) -> Result<(), PumpError<T::Err, C::Err, H::Err>>
where
    T: KvpTransport,
    C: KvpCodec,
    H: KvpHandler<C::Wire>,
{
    if let Some(bytes) = transport.recv().map_err(PumpError::Transport)? {
        let env = codec.decode(&bytes).map_err(PumpError::Codec)?;
        handler.on_message(env).map_err(PumpError::Handler)?;
    }
    Ok(())
}

#[derive(Debug)]
pub enum PumpError<TE: Debug, CE: Debug, HE: Debug> {
    Transport(TE),
    Codec(CE),
    Handler(HE),
}
```

```rust
#[inline]

    const Q: f32 = 1000.0;
    (x * Q).round() / Q
}

#[inline]
pub fn quantize_f64(x: f64) -> f64 {
    const Q: f64 = 1000.0;
    (x * Q).round() / Q
}

    v.sort_by_key(|x| f(x));
}

#[derive(Clone, Debug, Serialize, Deserialize, PartialEq)]
pub struct FullSnapshot {
    pub schema_version: SchemaVersion,
    pub tick: Tick,
    pub step_hash: String,
    pub world: WorldSnapshot,
    pub agents: Vec<AgentSnapshot>,
    pub items: Vec<ItemSnapshot>,
    pub events: Vec<EventSnapshot>,
    pub narrative_fragments: Vec<NarrativeFragment>,
}

impl Canonicalize for FullSnapshot {
    fn canonicalize(&mut self) {
        self.world.canonicalize();

        sort_by_key_stable(&mut self.agents, |a| a.agent_id);
        for a in &mut self.agents {
            a.canonicalize();
        }

        sort_by_key_stable(&mut self.items, |i| i.item_id);
        for i in &mut self.items {
            i.canonicalize();
        }

        // events stable by (tick, event_id)
        self.events.sort_by_key(|e| (e.tick, e.event_id));
        for e in &mut self.events {
            e.canonicalize();
        }

        // narrative stable by (entity_id, kind, text)
        self.narrative_fragments
            .sort_by(|a, b| (a.entity_id, a.kind, &a.text).cmp(&(b.entity_id, b.kind, &b.text)));
        for n in &mut self.narrative_fragments {
            n.canonicalize();
        }
    }
}

#[derive(Clone, Debug, Serialize, Deserialize, PartialEq)]
pub struct DiffBody {
    pub agent_moves: Vec<AgentMove>,
    pub agent_spawns: Vec<AgentSnapshot>,
    pub agent_despawns: Vec<i32>, // agent_id list

    // Phase 1 policy: replace-lists for rooms/events/narrative
    pub room_replacements: Vec<RoomSnapshot>,
    pub event_replacements: Vec<EventSnapshot>,
    pub narrative_replacements: Vec<NarrativeFragment>,
}

impl Canonicalize for DiffBody {
    fn canonicalize(&mut self) {
        sort_by_key_stable(&mut self.agent_moves, |m| m.agent_id);
        for m in &mut self.agent_moves {
            m.canonicalize();
        }

        sort_by_key_stable(&mut self.agent_spawns, |a| a.agent_id);
        for a in &mut self.agent_spawns {
            a.canonicalize();
        }

        self.agent_despawns.sort();

        sort_by_key_stable(&mut self.room_replacements, |r| r.room_id);
        for r in &mut self.room_replacements {
            r.canonicalize();
        }

        self.event_replacements.sort_by_key(|e| (e.tick, e.event_id));
        for e in &mut self.event_replacements {
            e.canonicalize();
        }

        self.narrative_replacements
            .sort_by(|a, b| (a.entity_id, a.kind, &a.text).cmp(&(b.entity_id, b.kind, &b.text)));
        for n in &mut self.narrative_replacements {
            n.canonicalize();
        }
    }
}

/// Convenience: build an envelope

pub fn make_envelope(payload: WirePayload, msg_id: String, sent_at_ms: u64) -> Envelope {
    let msg_type = match &payload {
        WirePayload::ViewerHello(_) => MsgType::ViewerHello,
        WirePayload::KernelHello(_) => MsgType::KernelHello,
        WirePayload::Subscribe(_) => MsgType::Subscribe,
        WirePayload::Subscribed(_) => MsgType::Subscribed,
        WirePayload::Unsubscribe(_) => MsgType::Unsubscribe,
        WirePayload::Ping(_) => MsgType::Ping,
        WirePayload::Pong(_) => MsgType::Pong,

        WirePayload::FullSnapshot(_) => MsgType::FullSnapshot,
        WirePayload::FrameDiff(_) => MsgType::FrameDiff,

        WirePayload::InputCommand(_) => MsgType::InputCommand,
        WirePayload::CommandAccepted(_) => MsgType::CommandAccepted,
        WirePayload::CommandRejected(_) => MsgType::CommandRejected,

        WirePayload::ReplayBegin(_) => MsgType::ReplayBegin,
        WirePayload::ReplayReady(_) => MsgType::ReplayReady,
        WirePayload::ReplaySeek(_) => MsgType::ReplaySeek,
        WirePayload::ReplayEnd(_) => MsgType::ReplayEnd,

        WirePayload::DebugProbe(_) => MsgType::DebugProbe,
        WirePayload::DebugProbeResult(_) => MsgType::DebugProbeResult,
        WirePayload::DesyncReport(_) => MsgType::DesyncReport,
        WirePayload::DesyncConfirmed(_) => MsgType::DesyncConfirmed,
        WirePayload::DesyncDenied(_) => MsgType::DesyncDenied,

        WirePayload::Warn(_) => MsgType::Warn,
        WirePayload::Error(_) => MsgType::Error,
    };

    Envelope {
        kvp_version: KVP_VERSION_V0_1.to_string(),
        msg_type,
        msg_id,
        sent_at_ms,
        payload,
    }
}
```
